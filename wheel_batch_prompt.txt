一、整体结构
1. 如果需要预先声明变量，必须先写一个顶格变量块“【变量】”。变量块和轮盘块同级，不属于任何轮盘，通常放在所有“【轮盘：...】”之前。普通变量可写初始值，计算变量可写计算公式。没有变量需求时不要写变量块。
2. 每个轮盘必须从顶格轮盘标记开始，格式是“【轮盘：轮盘名】”。例如“【轮盘：世界观】”。这里的轮盘名就是轮盘唯一 ID 和显示名。只有这种格式会被识别为轮盘，普通的“选项:”不能作为轮盘标题。
3. 变量块和轮盘标记都是最外层级，多批内容可以直接上下拼接，不需要再包一层总结构。禁止把“【变量】”写在某个轮盘下面，禁止把“计算公式”写成轮盘字段。
4. 轮盘下面每个属性缩进两个空格。可用属性：分类、说明、默认跳转、全部选项单独跳转、允许重复抽取、数值变化、条件跳转、选项。
5. “选项:”下面每个选项缩进四个空格，并以“- ”开头。选项字段用“ | ”分隔。
6. 选项名就是选项的唯一 ID 和显示名，不需要额外写变量。
7. 没有写出的字段就使用默认值。不要为了凑字段写“无”“未设置”“空”。

一补充、变量块字段
1. 变量块格式：
【变量】
  - 变量名 | 初始值: 100
  - 计算变量名 | 计算公式: 战斗力*0.1
  - 隐藏变量名 | 初始值: 0 | 隐藏: 是
2. 变量块是全局配置，不是轮盘配置；它只负责声明变量，不负责触发变化。变量真正发生变化的位置，是轮盘字段“数值变化”或选项字段“数值变化”。
3. 普通变量可以被“数值变化”直接增减或设值，例如“数值变化: 战斗力+10”。
4. 计算变量没有自己的固定值，不能作为“数值变化”的目标。它会根据其他普通变量的当前值实时计算，所以当选项或轮盘导致普通变量变化时，计算变量也会自动变化。
5. 计算变量必须写在顶格“【变量】”块里，例如“  - 存活概率 | 计算公式: 战斗力*0.1”。不要写成“计算公式: ...”这种轮盘字段，也不要写在“选项:”下面。
6. 计算变量可以用于权重公式、条件跳转和 Prompt 变量，例如 {{stat:存活概率}}。
7. 计算公式支持变量名、数字、+ - * / ^、括号、x 或 × 乘号，以及 min/max/abs/round/floor/ceil/clamp。
8. 隐藏变量不会显示在最终数值记录里，但仍可用于公式、条件和 Prompt 指定变量。
9. 普通变量如果只是被“数值变化”使用，可以不提前声明，系统会自动创建。例如“数值变化: 金钱-1”会自动创建“金钱”变量，初始值为 0。
10. 但只要是公式变量/计算变量，就不能依靠“数值变化”自动创建，必须在顶格“【变量】”块中提前声明。

一补充2、公式变量（计算变量）硬性规则
1. 公式变量也叫计算变量，它必须写在顶格“【变量】”块中，和所有“【轮盘：...】”同级。
2. 公式变量不属于任何单个轮盘，也不属于任何单个选项；不要把公式变量写进轮盘字段、选项字段或“选项:”列表里。
3. 公式变量没有自己的初始值，也不会被“数值变化”直接修改。它的值来自其他普通变量的实时计算。
4. 轮盘和选项只能通过“数值变化”改变普通变量；普通变量变化后，公式变量会自动跟着变化。
5. 公式变量可以引用普通变量，也可以引用其他公式变量，但不能循环引用。
6. 公式变量可以用于“权重公式”和“条件跳转”，例如“权重公式: 存活概率”或“条件跳转: 存活概率<5 -> 死亡结局”。
7. 错误写法：在“【轮盘：战斗】”下面写“计算公式: 战斗力*0.1”。
8. 错误写法：在选项里写“- 存活概率 | 计算公式: 战斗力*0.1”。
9. 正确写法：先在顶格“【变量】”里声明“存活概率”，再在轮盘或选项里通过“数值变化”改变“战斗力”。
10. 正确示例：
【变量】
  - 战斗力 | 初始值: 100
  - 存活概率 | 计算公式: 战斗力*0.1

【轮盘：战斗】
  选项:
    - 受伤 | 数值变化: 战斗力-20
    - 爆发 | 数值变化: 战斗力+10

二、轮盘字段
1. 分类: 用于编辑器里分组管理，例如“分类: 开局”。不写则为未分类。
2. 说明: 解释这个轮盘的作用，例如“说明: 决定角色所在的世界。”不写则为空。
3. 默认跳转: 轮盘完成后，在选项没有单独跳转时走这里。例如“默认跳转: 身份”。不写默认跳转或写“默认跳转: 结束”，都表示流程结束。
4. 全部选项单独跳转: 是 表示这个轮盘不使用默认跳转，也不使用轮盘级条件跳转；每个选项都按自己的“跳转”或“条件跳转”决定去向。没有目标的选项会结束流程。
5. 允许重复抽取: 是/否。写“允许重复抽取: 否”表示这个轮盘在同一次流程里不会重复抽到已经出现过的选项；不写或写“是”表示允许重复。
6. 数值变化: 轮盘级变量变化，在进入并执行这个轮盘时应用一次。例如“数值变化: 生命-1; 压力+2”。不写则没有轮盘级数值变化。
7. 条件跳转: 轮盘级条件跳转，只在没有开启“全部选项单独跳转”时有效。它会先判断条件，命中则跳到目标；都不命中才走默认跳转。例如“条件跳转: 压力>5 -> 崩溃事件”。可用分号写多条。
8. 批量文本不设置起始轮盘。用户已有的起始轮盘会保持不变，所以不要输出“起始: 是”。

三、选项字段
1. 基本格式：- 选项名 | 权重: 2 | 说明: 文本 | 跳转: 目标轮盘 | 数值变化: 变量+1 | 条件跳转: 条件 -> 目标轮盘
2. 选项名必填。权重不写时默认为 1。
3. 权重: 必须是大于 0 的数字，最多两位小数，数字越大越容易抽中。例如“权重: 3”或“权重: 0.25”。不写就是 1。
4. 权重公式: 可引用变量并使用 + - * / ^、括号和 min/max/abs/round/floor/ceil/clamp。例如“权重公式: 战斗力x0.1”或“权重公式: max(0.01, (100-战斗力)*0.1)”。只有确实需要动态概率时才写。
5. 说明: 命中这个选项后的补充说明。不写则为空。
6. 跳转: 选项单独跳转。写了“跳转: 目标轮盘”时，这个选项会覆盖轮盘默认跳转。写“跳转: 结束”或不写跳转，表示不单独指定目标。
7. 数值变化: 命中这个选项后应用的变量变化。例如“数值变化: 金钱-1; 声望+2; 阵营=1”。不写则没有选项级数值变化。
8. 条件跳转: 选项级条件跳转，只有这个选项被命中后才判断。它优先于该选项的普通“跳转”；都不命中时才走这个选项的普通“跳转”。例如“条件跳转: 压力>5 -> 崩溃事件”。可用分号写多条。

四、跳转关系
1. 轮盘默认跳转是主流程：所有没有单独跳转的选项都会走它。
2. 轮盘条件跳转是轮盘级分支：抽完选项并应用数值变化后，先判断轮盘条件，命中则跳转；不命中才走默认跳转。
3. 选项跳转是结果级分支：只有抽到该选项时生效，会覆盖轮盘默认跳转。
4. 选项条件跳转是更细的结果级条件分支：只有抽到该选项时判断，命中则跳转；不命中才走该选项的普通跳转。
5. 跳转目标直接写轮盘名，不要写 ID。目标不存在时系统会提醒用户创建空轮盘、改为结束或返回修改。

五、数值变化与条件
1. 数值变化可写：变量+1、变量-2、变量=0。也兼容以计算符号开头的写法：+变量+1、-变量-2、=变量=0。
2. 多个数值变化用分号分隔，例如“数值变化: 生命-1; 压力+2”。
3. 条件可写：变量>5、变量<3、变量=1、1<=变量<=5。
4. 多个条件同时满足用“且”连接，例如“压力>5 且 金钱<2 -> 逃亡事件”。
5. 多条条件跳转用分号分隔，按书写顺序判断。
6. 条件里的变量可以是普通变量，也可以是计算变量。例如“存活概率>8 -> 高危战斗”。

八、额外备注
1. 数值的主要用法：没有必要每一处都加，除非要求数据化角色（例如力量敏捷），但是像多次抽取，可以使用一个xx抽取次数的数值，和每次抽取-1，大于0重新抽取的逻辑来做，这种情况可以用轮盘级数值变化和判断来做，减少资源使用。还有，如果有的内容是要先进入从分线进入总线，再进入对应的分线，那也可以用数值来标记之前的分线路径 其中如果是选中了就跳转，那么就不要用数值，直接用选项跳转即可、
2. 分线跳转：如果你想让在某一个轮盘结束后，开始分线，那么需要让这个选项直接做条件跳转，而不是再建立一个中间轮盘去做跳转，禁止出现空轮盘
3. 轮盘名作为唯一ID，必须严格对齐，前后一致，尤其是当生成内容过长，需要跨多轮对话时，一定要保证之前的跳转名和之后的轮盘名一致
4. 轮盘和选项的说明，主要用途是辅助AI生成，次要是让玩家了解，不是用于解释轮盘逻辑的
5. 权重不是一定要加的，只在稀有度有区别的地方加权重，大部分内容都可以保持默认权重（即全1），这样可以让轮盘看起来更舒服，如果加入了大量非必要权重，轮盘会非常不规整
6. 尽可能避免使用没有或者仅有一个选项的轮盘作为跳转，跳转直接在上一个有内容的轮盘添加即可，除非是概率判断，否则不建议增加专用的判断/跳转轮盘
